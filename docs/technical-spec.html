Implementation Details & Technical Requirements

Backend Implementation

FastAPI Server

Framework: FastAPI 0.104+

Async Runtime: Uvicorn

Port: 8000 (default)

Features: REST API, WebSocket, automatic documentation

Python Execution Sandbox

Requirement: Execute generated Python code in isolated environment

Implementation: subprocess module with timeout

Timeout: 5 minutes (configurable)

Memory Limit: 4GB (configurable)

Output Capture: stdout, stderr, return code

State Management

RunManager Class: Manages pipeline run state

Persistence: JSON files in runs/ directory

Checkpoint Interval: After each phase

Recovery: Resume from last checkpoint

Model Factory

Supported Providers:

  • Ollama (local, default)
  • AWS Bedrock
  • OpenAI
  • Lemonade/llama.cpp

Configuration: Environment variables or config file

Frontend Implementation

Next.js Application

Framework: Next.js 14+

Language: TypeScript

Styling: Tailwind CSS

Port: 3000 (default)

API Client Service

Location: frontend/src/services/api.ts

Features:

  • REST API calls with error handling
  • WebSocket connection management
  • Automatic reconnection with exponential backoff
  • Request/response interceptors

Real-Time Updates

WebSocket Connection: /ws/pipeline/{run_id}

Message Types:

  • agent_update: Agent status changes
  • phase_update: Phase transitions
  • log_entry: New log messages
  • score_update: Performance score changes

Agent Implementation

Agent Base Class

class Agent: def __init__(self, model_provider: str, model_id: str): self.model = create_model(model_provider, model_id) async def execute(self, input_data: dict) -> dict: prompt = self.build_prompt(input_data) response = await self.model.generate(prompt) return self.parse_response(response)

Prompt Engineering

Prompt Structure:

  • System message: Agent role and responsibilities
  • Context: Task description and data
  • Instructions: Specific task requirements
  • Examples: Few-shot examples
  • Output format: XML-style tags for parsing

Error Handling

Retry Logic: Exponential backoff (max 3 attempts)

Fallback: Use cached results or default values

Logging: Comprehensive error logging

Data Models

TaskDescription

class TaskDescription(BaseModel): description: str task_type: str # classification, regression, etc. data_modality: str # tabular, image, text evaluation_metric: str dataset_path: str train_file: Optional[str] = None test_file: Optional[str] = None

PipelineRun

class PipelineRun(BaseModel): run_id: str task: TaskDescription status: str # pending, running, completed, failed phase: int # 1, 2, 3 agents: Dict[str, AgentStatus] scores: Dict[str, float] created_at: datetime updated_at: datetime

Database Schema

File-Based Storage

Directory Structure:

runs/ {run_id}/ config.json # Task configuration state.json # Pipeline state phase1/ initial_solution.py scores.json phase2/ refinement_1.py ablation_results.json phase3/ ensemble.py submission.csv

Security Implementation

Input Validation

Requirement: Validate all API inputs using Pydantic models

Code Execution Sandbox

Requirement: Execute generated code in isolated subprocess with resource limits

Secret Management

Requirement: Use environment variables for API keys, no hardcoded secrets

Security Scanning

Components:

  • Secret detector: Finds hardcoded secrets
  • API auditor: Checks OWASP Top 10 compliance
  • Dependency analyzer: Scans for vulnerable packages

Performance Optimization

Caching

Web Search Cache: Cache identical queries for 24 hours

Model Cache: Cache LLM responses for identical prompts

Parallelization

Concurrent Evaluation: Evaluate multiple models in parallel

Async Operations: Use async/await for I/O operations

Resource Management

Memory Limits: 4GB per code execution

Timeout Limits: 5 minutes per code execution

Disk Cleanup: Remove old runs after 30 days

Deployment Configuration

# Backend Dockerfile FROM python:3.11-slim WORKDIR /app COPY requirements.txt . RUN pip install -r requirements.txt COPY src/ . CMD ["uvicorn", "mle_star.api.server:app", "--host", "0.0.0.0"]

Docker Compose

Services:

  • backend: FastAPI server
  • frontend: Next.js application
  • ollama: Local LLM (optional)

Monitoring & Logging

Structured Logging

Format: JSON with correlation IDs

Levels: DEBUG, INFO, WARNING, ERROR, CRITICAL

Output: Console and file

Metrics

Pipeline Metrics:

  • Execution time per phase
  • Success/failure rates
  • Agent latencies
  • Resource usage
